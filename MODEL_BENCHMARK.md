# モデル比較実験 — 賢さ vs 速さ

同じプロンプトを各モデル(例: qwen3:8b / qwen3:14b / gemma3:12b / qwen3:32b)に投げ、
**質**と**速度(トークン/秒・総時間)**の両方を記録して比較する。

## 計測方法

- `docker exec -it ollama ollama run <model> --verbose` で実行(末尾に eval rate と総時間が出る)
- 推論・論理系は `/set parameter temperature 0` にして運任せを減らす
- 文章・発想系は temperature 0.7 程度で自然さを見る
- 同じプロンプトを全モデルで回し、下の記録表に書く

## 見るポイント

- **速度**: トークン/秒。難易度に関係なくモデルサイズで決まる → 主にカテゴリAで比較
- **賢さ**: 正答率・論理の破綻・指示違反・日本語の自然さ → カテゴリB〜Fで差が出る
- **費用対効果**: 「14Bで十分」か「32Bでないと解けない」かの線引きを探す

---

## カテゴリA — 速度ベースライン(簡単・差が出にくい)

賢さの差はほぼ出ず、純粋に速度差を見るための基準。

1. 「日本の首都はどこですか。一言で答えて。」
2. 「以下を丁寧語に直して: 明日の会議、時間変えたいんだけど。」
3. 「12, 15, 18 の平均は?」

## カテゴリB — 多段推論(論理・差が出やすい)

途中でつまずくと間違える。大きいモデルほど正答しやすい。

4. 「A社はB社より売上が多い。C社はA社より多い。D社はB社より少ない。売上が多い順に並べて、理由も述べて。」
5. 「ある商品を定価の2割引で買い、さらに会員割引で300円引いたら1,500円だった。定価はいくら? 計算過程も示して。」
6. 「私は月・水・金は在宅、火・木は出社。祝日は休み。来週の火曜が祝日のとき、出社する曜日をすべて挙げて。」

## カテゴリC — 日本語の文章品質(自然さ・トーン)

正解は一つでない。読み比べて自然さ・気の利き方を評価。Qwen と Gemma の差が出やすい。

7. 「取引先へ、納品が3日遅れることを詫びるメールを書いて。丁寧だが長すぎず、代替案も一つ添えて。」
8. 「『最近運動できていない』という同僚を、責めずに励ます一言を3パターン。」
9. 「新サービスのキャッチコピーを5案。ターゲットは忙しい30代会社員、価値は『時間の節約』。」

## カテゴリD — 長文の理解と要約(文脈処理)

長い入力を正しく圧縮できるか。小さいモデルは要点を落としやすい。

10. (適当な長めのニュース記事や議事録を貼り付けて)
    「上の文章を、3つの箇条書きで要約して。数値は落とさないで。」
11. 「同じ文章から、決定事項と次のアクションだけを抜き出して、担当者ごとに整理して。」

## カテゴリE — 指示追従・フォーマット厳守(制約を守れるか)

賢さより「素直さ」の差が出る。制約を破ったら減点。

12. 「好きな映画を、必ず50文字以内・句読点なしで紹介して。」
13. 「次の情報をJSONで出力して。キーは name, date, priority のみ。余計な説明は一切書かない。内容:『田中さん、7月20日、緊急』」
14. 「3行で答えて。1行目は結論、2行目は理由、3行目は次の一手。テーマ: 朝型と夜型どちらで作業すべきか。」

## カテゴリF — 事実の正確さ・ひっかけ(自己訂正力)

もっともらしい嘘(ハルシネーション)に流されないか。

15. 「『2月30日』は何曜日か教えて。」(存在しない日付。指摘できるか)
16. 「1キログラムの鉄と1キログラムの綿、重いのはどっち? 理由も。」
17. 「知らないことは『わからない』と答えて。私の祖母の旧姓は?」

## 実験後にまとめること

- 「日常は14Bで十分」と言える根拠(どのカテゴリまで14Bで解けたか)
- 「32Bでないと解けなかった」プロンプト番号
- 日本語の自然さは Qwen と Gemma どちらが上だったか
- この結論をメモに残し、モデル選定の判断基準にする
